# Ideation Techniques Reference

## Analogous Inspiration

Look at how other industries solve structurally similar problems.

### How to Find Analogies
1. Abstract the problem: "We need to [verb] [noun] under [constraint]"
2. Ask: "What other domains face this same abstract challenge?"
3. Research how those domains solve it
4. Translate the solution principle back to your domain

### Example Abstractions
- "Coordinate many agents with limited communication" — ant colonies, air traffic control, distributed databases
- "Build trust with strangers quickly" — Airbnb reviews, medical credentials, open-source contribution history
- "Handle unpredictable demand spikes" — emergency rooms, cloud auto-scaling, restaurant reservations

### Quality Check
Good analogies:
- Come from a genuinely different domain (not a direct competitor)
- Share structural similarity, not surface similarity
- Offer a transferable principle, not a feature to copy

## First Principles Thinking

### The Process
1. **List everything "everyone knows"** about this problem space — the conventional wisdom
2. **Challenge each item:** Is this a physical law, or just a convention?
   - Physical law: "Users need internet to sync data" — Actually, CRDTs enable offline-first
   - Convention: "This needs a mobile app" — Maybe a text message works better
3. **Keep only the unchallengeable constraints**
4. **Rebuild from those constraints alone** — ignore everything else

### Forcing Questions
- "What if this had never been done before — what would we build?"
- "What are we doing because competitors do it, not because users need it?"
- "What would we remove if we started over?"
- "What's the simplest possible version that delivers the core value?"

## 10x Radical Thinking

### The Process
1. **Go maximally ambitious:** "What if this was 10x better than anything that exists?"
2. **Remove all constraints:** Money, time, technology, regulations — ignore them all
3. **Describe the radical vision** in detail
4. **Extract the kernel:** What's the core insight of that 10x idea?
5. **Scale back to 1x:** How could you deliver that kernel with real-world constraints?

### Forcing Questions
- "What if this was free for everyone?"
- "What if this worked instantly, with zero learning curve?"
- "What if this solved the problem so well it made the problem impossible to have?"
- "What would Google/Apple/the best company in the world build?"
- "What would this look like in 10 years?"

## General Ideation Rules

- **Quantity over quality** — aim for volume first, filter later
- **No self-censoring** — every idea gets written down
- **Build on ideas** — "what if we combined X and Y?"
- **Stay specific** — "an app that..." is weak; "a notification that triggers when [specific condition] and shows [specific info]" is strong
- **Reference empathy data** — the best ideas solve real, observed needs
